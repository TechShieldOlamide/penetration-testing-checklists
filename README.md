# Penetration Testing Checklists

Structured penetration testing checklists for web applications, APIs, networks, and mobile apps — built for security consultants and compliance teams.

## What is Penetration Testing?

Penetration testing is the practice of simulating real-world attacks on systems, applications, and networks to identify security vulnerabilities before malicious actors do. These checklists provide a structured methodology for conducting security assessments and documenting findings.

| Checklist | Description |

| API Pentest Checklist | Security testing checklist for REST and GraphQL APIs |
| Network Pentest Checklist | Network infrastructure security assessment guide |

## How to Use

1. Select the checklist relevant to your engagement
2. Work through each test case systematically
3. Document findings using the pentest report template
4. Classify findings by severity
5. Deliver report with remediation recommendations

## Severity Ratings

| Rating | Description | Response Time |
|---|---|---|
| Critical | Immediate exploitation risk, high business impact | Immediate |
| High | Significant risk, requires urgent attention | Within 48 hours |
| Medium | Moderate risk, should be addressed | Within 30 days |
| Low | Minor risk, low business impact | Next maintenance cycle |
| Informational | No direct risk, best practice recommendation | As appropriate |

## Disclaimer

These checklists are for authorized security testing only. Always obtain written permission before conducting any security assessment. Unauthorized testing is illegal and unethical.
